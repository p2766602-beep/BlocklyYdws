# 一、12 個 SmartRing 動畫密室資料

## 1. 全亮圖樣密室

示範積木：示範圖樣：全亮，顏色紅色  
核心概念：整體輸出、顏色設定、穩定狀態  
難度：L1  
目標動畫：全部 LED 同時顯示指定顏色，形成穩定的全亮畫面。  
核心通關句：全部 LED 同時顯示指定顏色，形成穩定的起始畫面。  

第 1 關必要觀察：
- 全部 LED 同時亮起。
- 顏色是指定顏色。
- 狀態穩定，不是移動或閃爍。

第 2 關必要規則：
- 清除或設定初始狀態。
- 從第 1 顆到第 12 顆 LED 都設定為指定顏色。
- 顯示暫存陣列到 SmartRing。

第 3 關流程骨架：
```text
開始
清除暫存陣列
從 1 到 12 計數：
  設定暫存陣列第 i 顆 LED 顏色為指定顏色
顯示暫存陣列到 SmartRing
結束
```

第 3 關流程卡範例：
A. 顯示暫存陣列到 SmartRing  
B. 從第 1 顆到第 12 顆 LED，全部設定為指定顏色  
C. 清除暫存陣列  
D. 開始  
E. 結束  

正確順序：D → C → B → A → E

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、從 1 到 12 計數、變數 i。  
第 5 關改造方向：改變顏色、開場畫面、完成提示燈、警示背景色。

## 2. 狀態顯示密室

示範積木：示範狀態顯示：分數，值 50，最大 100，顏色紅色  
核心概念：數值比例、資料視覺化、狀態顯示  
難度：L2  
目標動畫：將目前數值與最大值轉換成比例，再用亮燈數量呈現狀態。  
核心通關句：把數值換算成比例，再用亮燈數量顯示目前狀態。  

第 1 關必要觀察：
- 不是全部 LED 都亮。
- 亮燈數量代表目前數值比例。
- 顏色是指定顏色。
- 顯示像進度條、分數條或能量條。

第 2 關必要規則：
- 先取得目前值與最大值。
- 將目前值除以最大值，換算成比例。
- 把比例轉成亮燈顆數。
- 從第 1 顆到亮燈顆數設定顏色。
- 顯示暫存陣列。

第 3 關流程骨架：
```text
開始
設定 value 與 max
計算亮燈數量 = value / max * 12
清除暫存陣列
從 1 到亮燈數量計數：
  設定暫存陣列第 i 顆 LED 顏色為指定顏色
顯示暫存陣列到 SmartRing
結束
```

第 3 關流程卡範例：
A. 計算亮燈數量  
B. 顯示暫存陣列到 SmartRing  
C. 從第 1 顆到亮燈數量設定顏色  
D. 清除暫存陣列  
E. 設定 value 與 max  
F. 開始  
G. 結束  

正確順序：F → E → A → D → C → B → G

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、變數 value、變數 max、數學比例運算、從 1 到亮燈數量計數。  
第 5 關改造方向：分數條、血量條、能量條、進度條、倒數條。

## 3. 閃爍動畫密室

示範積木：播放閃爍動畫：顏色紅色，次數 3  
核心概念：亮滅切換、等待、重複結構、次數控制、暫存陣列顯示  
難度：L2  
目標動畫：全部 LED 同時亮紅色，等待一小段時間後全部熄滅，再等待一小段時間，以上流程重複 3 次。  
核心通關句：全部一起閃，亮滅都要等待，重複三次完成流程。  

第 1 關必要觀察：
- 全部 LED 一起變化。
- 顏色是紅色。
- 亮起後會熄滅，形成閃爍。
- 重複 3 次。
- 最後通常熄滅。

第 2 關必要規則：
- 全部 LED 亮紅色。
- 等待。
- 全部 LED 熄滅。
- 再等待。
- 重複 3 次。

第 3 關流程骨架：
```text
開始
重複 3 次：
  清除暫存陣列
  從 1 到 12 計數：
    設定暫存陣列第 i 顆 LED 顏色為紅色
  顯示暫存陣列到 SmartRing
  等待一段時間

  清除暫存陣列
  顯示暫存陣列到 SmartRing
  等待一段時間
結束
```

第 3 關流程卡範例：
A. 顯示暫存陣列到 SmartRing  
B. 等待一段時間  
C. 清除暫存陣列  
D. 從第 1 顆到第 12 顆 LED，全部設定為紅色  
E. 重複 3 次開始  
F. 顯示暫存陣列到 SmartRing  
G. 等待一段時間  
H. 清除暫存陣列  
I. 重複 3 次結束  

正確順序：E → C → D → A → B → H → F → G → I

流程卡後理解問題：
為什麼這個流程中需要出現兩次「顯示暫存陣列到 SmartRing」？

可接受回答方向：
因為亮紅色要顯示一次，清除變暗也要再顯示一次。

核心通關問題：為什麼閃爍動畫不能只寫「亮紅色」，還需要「清除暫存陣列」、「顯示暫存陣列」與「等待」？  
通過條件：學生能說出清除會形成暗的狀態、顯示暫存陣列才會更新畫面、等待讓人看得出亮與暗。  

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色為紅色、顯示暫存陣列到 SmartRing、等待 N 毫秒、重複 N 次、從 1 到 12 計數。  
第 5 關改造方向：改顏色、改次數、改速度、雙色警示燈、倒數提示燈。

## 6. 跑馬燈動畫密室

示範積木：播放跑馬燈動畫：顏色紅色，速度 100  
核心概念：單點移動、位置變數、迴圈  
難度：L3  
目標動畫：同一時間只有一顆 LED 亮起，亮點依序移動，形成跑馬燈效果。  
核心通關句：每次只亮目前位置，清除後移到下一顆，形成移動效果。  

第 1 關必要觀察：
- 同一時間主要只有一顆 LED 亮。
- 亮點會移動。
- 移動方向有順序。
- 每一步有等待。
- 與左移／右移動畫不同：跑馬燈是單顆亮點移動，不是四顆連續圖樣整段位移。

第 2 關必要規則：
- 清除暫存陣列。
- 設定目前位置 LED。
- 顯示暫存陣列。
- 等待。
- 目前位置往下一顆移動。
- 重複直到跑完。

第 3 關流程骨架：
```text
開始
從 position = 1 到 12：
  清除暫存陣列
  設定暫存陣列第 position 顆 LED 顏色為紅色
  顯示暫存陣列到 SmartRing
  等待一段時間
結束
```

第 3 關流程卡範例：
A. 設定暫存陣列第 position 顆 LED 為紅色  
B. 顯示暫存陣列到 SmartRing  
C. 從 position = 1 到 12 計數  
D. 清除暫存陣列  
E. 等待一段時間  
F. 開始  
G. 結束  

正確順序：F → C → D → A → B → E → G

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、等待 N 毫秒、變數 position、從 1 到 12 計數、重複 N 次。  
第 5 關改造方向：改速度、改顏色、兩顆燈一起移動、按鈕控制方向。

## 7. 左移動畫密室

示範積木：播放左移動畫：顏色紅色，次數 1，速度 100  
核心概念：連續圖樣位移、起始位置變數、陣列位移概念、邊界處理  
難度：L4  
目標動畫：一開始第 5、6、7、8 顆 LED 亮起，形成四顆連續圖樣。接著這四顆連續圖樣整段往左移動。  
核心通關句：第 5～8 顆形成連續圖樣，再用起始位置變數讓整段圖樣往左位移。  

目標變化範例：
```text
初始：5、6、7、8
左移 1：4、5、6、7
左移 2：3、4、5、6
左移 3：2、3、4、5
左移 4：1、2、3、4
```

第 1 關必要觀察：
- 一開始不是單顆 LED，而是第 5、6、7、8 顆四顆連續 LED 同時亮起。
- 亮起的是一整段圖樣。
- 這整段圖樣會一起往左移動。
- 每次移動後，仍維持四顆連續亮燈。
- 到達左側邊界時，第 1、2、3、4 顆亮起。
- 與跑馬燈不同：跑馬燈是單顆亮點移動，左移動畫是四顆連續圖樣整段位移。

第 2 關必要規則：
- 設定起始位置 startPosition = 5。
- 設定圖樣長度 patternLength = 4。
- 每一格畫面都先清除暫存陣列。
- 從 i = 0 到 patternLength - 1，設定第 startPosition + i 顆 LED 為指定顏色。
- 顯示暫存陣列到 SmartRing。
- 等待。
- startPosition = startPosition - 1，讓圖樣往左移動。
- 需要注意 startPosition 不應小於 1，這是邊界條件。

第 3 關流程骨架：
```text
開始
設定 startPosition = 5
設定 patternLength = 4
重複指定次數或直到 startPosition = 1：
  清除暫存陣列
  從 i = 0 到 patternLength - 1：
    設定暫存陣列第 startPosition + i 顆 LED 顏色為紅色
  顯示暫存陣列到 SmartRing
  等待一段時間
  startPosition = startPosition - 1
結束
```

第 3 關流程卡範例：
A. 顯示暫存陣列到 SmartRing  
B. 設定 startPosition = 5，patternLength = 4  
C. 清除暫存陣列  
D. 從 i = 0 到 patternLength - 1 計數  
E. 設定暫存陣列第 startPosition + i 顆 LED 為紅色  
F. 等待一段時間  
G. startPosition = startPosition - 1  
H. 重複指定次數或直到 startPosition = 1  
I. 開始  
J. 結束  

正確順序：I → B → H → C → D → E → A → F → G → J

流程卡後理解問題：
為什麼左移動畫不能只用一個 position 代表單顆 LED，而要用 startPosition 和 patternLength？

可接受回答方向：
因為左移動畫移動的是四顆連續圖樣，不是單顆亮點；startPosition 代表整段圖樣的起點，patternLength 代表這段圖樣的長度。

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、等待 N 毫秒、變數 startPosition、變數 patternLength、變數 i、從 0 到 patternLength - 1 計數、startPosition - 1、邊界判斷。  
第 5 關改造方向：多次左移、循環左移、改變圖樣長度、左移後補新顏色、速度逐漸加快。

## 8. 右移動畫密室

示範積木：播放右移動畫：顏色紅色，次數 1，速度 100  
核心概念：連續圖樣位移、起始位置變數、陣列位移概念、右移規則、邊界處理  
難度：L4  
目標動畫：一開始第 5、6、7、8 顆 LED 亮起，形成四顆連續圖樣。接著這四顆連續圖樣整段往右移動。  
核心通關句：第 5～8 顆形成連續圖樣，再用起始位置變數讓整段圖樣往右位移。  

目標變化範例：
```text
初始：5、6、7、8
右移 1：6、7、8、9
右移 2：7、8、9、10
右移 3：8、9、10、11
右移 4：9、10、11、12
```

第 1 關必要觀察：
- 一開始不是單顆 LED，而是第 5、6、7、8 顆四顆連續 LED 同時亮起。
- 亮起的是一整段圖樣。
- 這整段圖樣會一起往右移動。
- 每次移動後，仍維持四顆連續亮燈。
- 到達右側邊界時，第 9、10、11、12 顆亮起。
- 與跑馬燈不同：跑馬燈是單顆亮點移動，右移動畫是四顆連續圖樣整段位移。

第 2 關必要規則：
- 設定起始位置 startPosition = 5。
- 設定圖樣長度 patternLength = 4。
- 每一格畫面都先清除暫存陣列。
- 從 i = 0 到 patternLength - 1，設定第 startPosition + i 顆 LED 為指定顏色。
- 顯示暫存陣列到 SmartRing。
- 等待。
- startPosition = startPosition + 1，讓圖樣往右移動。
- 需要注意 startPosition + patternLength - 1 不應大於 12，這是右側邊界條件。

第 3 關流程骨架：
```text
開始
設定 startPosition = 5
設定 patternLength = 4
重複指定次數或直到 startPosition + patternLength - 1 = 12：
  清除暫存陣列
  從 i = 0 到 patternLength - 1：
    設定暫存陣列第 startPosition + i 顆 LED 顏色為紅色
  顯示暫存陣列到 SmartRing
  等待一段時間
  startPosition = startPosition + 1
結束
```

第 3 關流程卡範例：
A. 顯示暫存陣列到 SmartRing  
B. 設定 startPosition = 5，patternLength = 4  
C. 清除暫存陣列  
D. 從 i = 0 到 patternLength - 1 計數  
E. 設定暫存陣列第 startPosition + i 顆 LED 為紅色  
F. 等待一段時間  
G. startPosition = startPosition + 1  
H. 重複指定次數或直到 startPosition + patternLength - 1 = 12  
I. 開始  
J. 結束  

正確順序：I → B → H → C → D → E → A → F → G → J

流程卡後理解問題：
為什麼右移動畫的邊界條件不是只看 startPosition，而是要看 startPosition + patternLength - 1？

可接受回答方向：
因為圖樣有四顆 LED，startPosition 只是第一顆的位置，最後一顆的位置是 startPosition + patternLength - 1，所以右移時要確認最後一顆沒有超過第 12 顆。

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、等待 N 毫秒、變數 startPosition、變數 patternLength、變數 i、從 0 到 patternLength - 1 計數、startPosition + 1、邊界判斷。  
第 5 關改造方向：多次右移、循環右移、改變圖樣長度、右移後補新顏色、將左移與右移整合成方向參數。

## 9. 來回移動動畫密室

示範積木：播放來回移動動畫：顏色紅色，次數 1，速度 100  
核心概念：方向切換、邊界判斷、選擇結構  
難度：L4  
目標動畫：亮點在 LED 上左右移動，碰到邊界後改變方向，形成來回移動效果。  
核心通關句：亮點碰到邊界後改變方向，形成左右來回移動。  

第 1 關必要觀察：
- 亮點會移動。
- 碰到邊界後會反方向移動。
- 需要方向切換。
- 每一步有等待。

第 2 關必要規則：
- 設定 position 與 direction。
- 清除暫存陣列。
- 設定目前 position 亮燈。
- 顯示暫存陣列。
- 等待。
- 若到邊界，改變 direction。
- 根據 direction 更新 position。

第 3 關流程骨架：
```text
開始
設定 position
設定 direction
重複指定次數：
  清除暫存陣列
  設定暫存陣列第 position 顆 LED 顏色為紅色
  顯示暫存陣列到 SmartRing
  等待
  如果 position 到達左邊界或右邊界：
    direction 改變
  position = position + direction
結束
```

第 3 關流程卡範例：
A. 如果 position 到邊界，改變 direction  
B. 清除暫存陣列  
C. 設定 position 與 direction  
D. 顯示暫存陣列到 SmartRing  
E. 設定暫存陣列第 position 顆 LED 為紅色  
F. 等待  
G. position 依 direction 更新  
H. 重複指定次數  
I. 開始  
J. 結束  

正確順序：I → C → H → B → E → D → F → A → G → J

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、等待 N 毫秒、變數 position、變數 direction、如果／否則、邊界判斷、position 加 1 或減 1。  
第 5 關改造方向：彈跳球動畫、到邊界時變色、速度逐漸變快、兩個亮點同時來回移動。

## 10. 交錯閃爍動畫密室

示範積木：播放交錯閃爍動畫：顏色1 紅色，顏色2 紅色，次數 3  
核心概念：奇偶判斷、分組顯示、條件結構  
難度：L4  
目標動畫：依照 LED 編號奇偶分組，讓兩組 LED 以交錯方式顯示或閃爍。  
教學提醒：若顏色1與顏色2都為紅色，學生較難觀察，建議示範時改成紅色與藍色。  
核心通關句：依照 LED 編號奇偶分組，讓不同組交錯顯示。  

第 1 關必要觀察：
- LED 分成兩組。
- 常見分組是奇數與偶數。
- 兩組交錯顯示或交替閃爍。
- 若兩色不同，更容易觀察。

第 2 關必要規則：
- 從第 1 顆到第 12 顆檢查。
- 如果編號是奇數，設定顏色 1。
- 如果編號是偶數，設定顏色 2。
- 顯示暫存陣列。
- 等待。
- 可交換兩組顏色後再顯示。

第 3 關流程骨架：
```text
開始
重複指定次數：
  清除暫存陣列
  從 1 到 12：
    如果 i 是奇數：
      設定第 i 顆為顏色 1
    否則：
      設定第 i 顆為顏色 2
  顯示暫存陣列到 SmartRing
  等待

  清除暫存陣列
  從 1 到 12：
    如果 i 是奇數：
      設定第 i 顆為顏色 2
    否則：
      設定第 i 顆為顏色 1
  顯示暫存陣列到 SmartRing
  等待
結束
```

第 3 關流程卡範例：
A. 如果 i 是奇數設定顏色 1，否則設定顏色 2  
B. 顯示暫存陣列到 SmartRing 並等待  
C. 重複指定次數  
D. 清除暫存陣列  
E. 從 1 到 12 計數  
F. 第二次改成奇偶顏色交換  
G. 再次顯示並等待  
H. 開始  
I. 結束  

正確順序：H → C → D → E → A → B → D → E → F → G → I

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 顏色、顯示暫存陣列到 SmartRing、等待 N 毫秒、從 1 到 12 計數、變數 i、如果／否則、餘數運算、重複 N 次。  
第 5 關改造方向：紅藍交錯、奇偶交換、三色分組、節奏燈、警示燈。

## 11. 呼吸燈動畫密室

示範積木：播放呼吸燈動畫：顏色紫色，次數 1  
核心概念：亮度漸變、遞增遞減、巢狀迴圈  
難度：L5  
目標動畫：LED 亮度逐漸增加，再逐漸降低，形成像呼吸一樣的漸變效果。  
核心通關句：亮度逐漸增加再逐漸減少，形成像呼吸一樣的漸變。  

第 1 關必要觀察：
- 不是突然亮或突然暗。
- 亮度逐漸增加。
- 亮度再逐漸減少。
- 顏色大致維持同一色系。
- 有漸變節奏。

第 2 關必要規則：
- 設定 brightness 由小變大。
- 每個亮度都更新 12 顆 LED。
- 顯示暫存陣列並等待。
- 再讓 brightness 由大變小。
- 重複指定次數。

第 3 關流程骨架：
```text
開始
重複指定次數：
  從 brightness 小到大：
    清除暫存陣列
    從 1 到 12：
      依 brightness 設定第 i 顆 LED 的 RGB 數值
    顯示暫存陣列到 SmartRing
    等待

  從 brightness 大到小：
    清除暫存陣列
    從 1 到 12：
      依 brightness 設定第 i 顆 LED 的 RGB 數值
    顯示暫存陣列到 SmartRing
    等待
結束
```

第 3 關流程卡範例：
A. 從 brightness 小到大  
B. 從 brightness 大到小  
C. 依 brightness 設定 12 顆 LED 的 RGB  
D. 顯示暫存陣列到 SmartRing 並等待  
E. 重複指定次數  
F. 清除暫存陣列  
G. 開始  
H. 結束  

正確順序：G → E → A → F → C → D → B → F → C → D → H

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 紅色為／綠色為／藍色為、顯示暫存陣列到 SmartRing、等待 N 毫秒、變數 brightness、變數 i、從小到大計數、從大到小計數、巢狀迴圈。  
第 5 關改造方向：不同顏色呼吸燈、多次呼吸、心跳燈、按鈕觸發呼吸燈、亮度速度調整。

## 12. 彩虹動畫密室

示範積木：播放彩虹動畫：速度 60  
核心概念：顏色序列、循環位移、陣列與模組化  
難度：L5  
目標動畫：使用多個顏色依序排列在 LED 上，並透過顏色位置的循環位移形成彩虹流動效果。  
核心通關句：使用多個顏色依序排列，再循環位移形成彩虹流動。  

第 1 關必要觀察：
- 有多種顏色。
- 顏色依序排列。
- 顏色會流動或循環移動。
- 每一步有等待。

第 2 關必要規則：
- 建立顏色序列。
- 每一顆 LED 對應一個顏色索引。
- 每次顯示後改變 offset。
- 用取餘數讓顏色循環。
- 重複形成流動效果。

第 3 關流程骨架：
```text
開始
建立顏色清單
設定 offset = 0
重複指定次數：
  清除暫存陣列
  從 i = 1 到 12：
    colorIndex = (i + offset) 取餘數
    依 colorIndex 設定第 i 顆 LED 的 RGB
  顯示暫存陣列到 SmartRing
  等待
  offset 改變
結束
```

第 3 關流程卡範例：
A. offset 改變  
B. 顯示暫存陣列到 SmartRing 並等待  
C. 建立顏色清單  
D. 從 i = 1 到 12 設定每顆 LED 的顏色  
E. 用 i 與 offset 計算 colorIndex  
F. 設定 offset = 0  
G. 重複指定次數  
H. 清除暫存陣列  
I. 開始  
J. 結束  

正確順序：I → C → F → G → H → D → E → B → A → J

第 4 關核心積木：清除暫存陣列、設定暫存陣列第 N 顆 LED 紅色為／綠色為／藍色為、顯示暫存陣列到 SmartRing、等待 N 毫秒、顏色清單、RGB 清單、變數 i、變數 colorIndex、變數 offset、取餘數、重複、函式。  
第 5 關改造方向：自訂彩虹顏色順序、雙向彩虹、慢速彩虹、彩虹跑馬燈、排序視覺化延伸。

# 三、共同評量規準

## 第 1 關：觀察之眼

通過條件：

- 學生能說出至少 2 到 3 個具體觀察重點。
- 觀察重點可包含顏色、範圍、方向、順序、次數、亮滅、結束狀態。
- 不能只回答「會亮」、「會動」、「很漂亮」。

## 第 2 關：規則之門

通過條件：

- 學生能把動畫拆成小動作。
- 學生能說出小動作的順序。
- 若有重複，學生能說出重複單位。
- 若有等待、邊界、方向、條件，學生能指出其必要性。

## 第 3 關：流程之卷

第 3 關採雙層挑戰：

### 第一層：自由流程挑戰

通過條件：

- 學生能把規則改寫成步驟。
- 流程中有必要的 LED 暫存陣列操作。
- 流程中有「顯示暫存陣列到 SmartRing」。
- 動畫類任務通常需要等待。
- 有重複或計數需求時，流程要包含迴圈或計數。
- 有方向、邊界、奇偶、亮度等需求時，流程要包含變數或條件判斷。

若流程不完整，AI 先提示缺漏並請學生修正一次。

### 第二層：流程卡排序挑戰

進入條件：

- 學生第二次修正後仍不完整。
- 學生明確表示不會。
- 學生無法把觀察描述轉成流程。

通過條件：

- 學生能排出合理流程順序。
- 排序後能用一句話說明流程中的一個核心原因。
- 例如：為什麼要顯示暫存陣列？為什麼要等待？為什麼要做邊界判斷？為什麼左移／右移需要 startPosition 與 patternLength？

# 四、共同追問句庫

## 觀察不清楚時

- 你看到的是全部 LED 一起變化，還是一顆一顆變化？
- 顏色是固定的，還是會改變？
- 動畫有沒有重複？
- 每一步之間有沒有停一下？
- 最後 LED 是亮著還是熄滅？
- 這個動畫是單顆亮點移動，還是一段連續圖樣整段移動？

## 規則不完整時

- 這個動畫的「一次變化」包含哪些小動作？
- 哪一段動作會重複？
- 如果少了等待，動畫還看得清楚嗎？
- 如果少了清除或更新狀態，動畫會發生什麼問題？
- 這個動畫有沒有方向或邊界條件？
- 如果是左移或右移，一開始亮的是哪幾顆 LED？
- 左移或右移時，是一顆燈在移動，還是第 5～8 顆形成的圖樣整段移動？

## 自由流程不完整時

- 你的流程目前比較像觀察描述，能不能改成一步一步的指令？
- 你有把「顯示暫存陣列」放進流程嗎？
- 你有安排等待時間嗎？
- 哪一段需要重複？
- 如果要設定 12 顆 LED，要用什麼方式從第 1 顆走到第 12 顆？
- 如果位置會移動，position 要在哪裡更新？
- 如果碰到邊界，要不要改變方向？
- 如果要判斷奇數偶數，需要用到什麼條件？
- 如果亮度要變化，brightness 要怎麼改變？
- 如果要讓第 5～8 顆的連續圖樣整段移動，startPosition 應該怎麼改變？
- 如果圖樣有 4 顆 LED，為什麼不能只設定一顆 LED？

## 流程卡排序後理解追問

- 為什麼這個流程需要「顯示暫存陣列到 SmartRing」？
- 為什麼等待要放在顯示之後？
- 為什麼需要先清除暫存陣列？
- 哪一段是重複執行的部分？
- 哪一個變數負責控制位置？
- 哪一個條件負責處理邊界？
- 為什麼左移／右移動畫需要 startPosition？
- 為什麼左移／右移動畫需要 patternLength？
- 跑馬燈和左移／右移動畫最大的差別是什麼？
